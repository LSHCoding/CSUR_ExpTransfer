# 1. Introduction

Large language models are no longer confined to one-shot responses to user queries. They now drive autonomous agents that act in external environments—calling tools, navigating browsers, coordinating with other agents on complex workflows. Such LLM-based agents operate in a closed loop of perception, decision, execution, and observation, producing a steady stream of interaction records: reasoning chains, tool calls, environment observations, human feedback. These records are the raw material from which an agent improves its own decision-making, and can be formalized as a tuple $e = (c, a, o, f)$ that captures, at each step, the decision context, the agent's output, the environment's response, and an optional evaluative signal. The community has developed an array of mechanisms for turning this raw material into reusable form: memory summarization distills trajectories into natural-language reflections; skill induction converts successful runs into reusable code; soft-prompt training compresses experience into continuous latent representations; reward modeling crystallizes human preferences into a parametric evaluator; trajectory fine-tuning bakes curated successes into policy weights; RLHF folds evaluator signals into the policy's decision preferences.

Yet a researcher building a continually self-improving agent faces questions that cut across these mechanisms. A given successful trajectory—should it be distilled into a retrievable reflection note, or trained into a mountable soft prompt? To give an agent a new capability—is it better to fine-tune it directly into the weights, or keep it as an externally auditable memory entry? Answers are scattered across the memory, post-training, and related literatures, and the prevailing taxonomies—organized by component (Memory / Planning / Tool Use) or by technique (RAG / SFT / RLHF)—do not address these questions directly.

These mechanisms can be read as instances of a single operation: each transforms how experience is represented, and each occupies a different point in the trade-off space of interpretability, inference efficiency, and editability. Consider a single successful agent trajectory. The most direct option is to drop it into the prompt as a few-shot example—immediate reuse, but every invocation pays the context-window cost. Memory summarization rewrites the trajectory into a tighter reflection or rule that remains human-readable and editable, raising semantic density and cutting context substantially. Trajectory fine-tuning instead trains the same material into the weights: inference no longer pays any context cost, but the experience is now hard to inspect or edit in isolation. Memory summarization, skill induction, reward modeling, trajectory fine-tuning, self-generated training—all can be read as this operation instantiated between different source and target carriers. We call the operation **Experience Transformation**.

Different transformation pathways unfold between different carriers. We organize carriers by the level at which experience exists within the model architecture, giving three top-level classes:

- **Tokenized carriers**: experience that enters the model's forward pass as discrete token sequences. By degree of formalization, this splits into **Narrative** (natural-language reflections, summaries, and the like) and **Schematic** (code, workflows, knowledge graphs, and other structured artifacts).
- **Latent carriers**: experience in the form of continuous vectors or hidden states that participate directly in attention or hidden-state computation.
- **Parametric carriers**: experience absorbed implicitly into network weights. By functional role, this splits into **Policy** (the actor weights that generate actions) and **Evaluator** (the judge weights that score trajectories).

Transformations across these three carrier classes form the analytical spine of this survey. We consolidate mechanisms scattered across memory, skill induction, alignment, and self-training into seven base pathways:

- **Narrative Abstraction** (Narrative → Narrative): within-carrier distillation of raw trajectories into more compact natural-language forms—reflections, rules, summaries.
- **Schematic Formalization** (Narrative → Schematic): natural-language experience converted into code, workflows, knowledge graphs, or other structures that can be parsed, executed, or traversed.
- **Latent-Space Transformation** (Tokenized → Latent): discrete-token experience transformed into continuous vectors that feed the forward pass directly.
- **Evaluator Internalization** (Tokenized → Evaluator): trajectory experience crystallized as a parametric evaluator—reward model, process reward model, or verifier.
- **Policy Internalization** (Tokenized → Policy): trajectory experience crystallized as decision-making capacity in policy weights.
- **Evaluator-Driven Optimization** (Evaluator → Policy): evaluator signals absorbed as generation preferences in policy weights.
- **Parametric Externalization** (Parametric → Tokenized): implicit experience in weights externalized as fresh tokenized trajectories.

We additionally identify composite pipelines that chain several base pathways together, and treat them as a first-class object of analysis.

This framing fills a dimension that existing surveys leave uncovered. Surveys of agent architectures (e.g., Wang et al., 2024; Xi et al., 2025) decompose agents by system components—Memory, Planning, Tool Use—but the same class of transformation operation routinely crosses these component boundaries: trajectory fine-tuning, for instance, belongs neither to Memory nor to Planning, and has no natural home in the component view. Memory-systems surveys (e.g., Zhang et al., 2025) characterize storage, retrieval, and forgetting within Tokenized carriers in depth, but do not engage with transformation across carriers. Alignment surveys focus on the single Evaluator → Policy pathway and trace the technical lineage from RLHF through DPO and its descendants; process-reward-model surveys (e.g., Zheng et al., 2025) concentrate on the design and training of the Evaluator carrier itself. Each of these works offers vertical depth within a system architecture, a single carrier, or a single technical family—but none treats experience transformation across carriers as an analytical axis in its own right.
