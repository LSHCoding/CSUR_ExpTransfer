# §1 Introduction

The transition from static large language models (LLMs, VLMs) to autonomous agents marks a fundamental shift in how models interact with the world. A static model processes a fixed input and produces a single output; an agent, by contrast, operates within a continuous *experience loop*. At each decision step \(t\), the agent perceives a context \(c_t\), produces an action \(a_t\) spanning heterogeneous forms—natural language reasoning chains, tool calls, code generation, GUI operations, embodied motor commands—receives environmental observations \(o_t\), and optionally obtains evaluative feedback \(f_t\). These accumulated decision records, which we term *agent experience*, constitute the primary substrate from which agents can learn to make better decisions over time.

The most straightforward approach to leveraging past experience—storing raw interaction logs and retrieving them via mechanisms like retrieval-augmented generation (RAG)—proves increasingly brittle and expensive at scale. Context windows impose hard limits on how much history an agent can attend to at once. Retrieval latency slows real-time decision-making. And the noise inherent in individual episodes obstructs cross-task generalization: a single trajectory may succeed for accidental reasons or fail despite containing locally effective strategies. In response, the research community has developed a diverse landscape of mechanisms for storing, transforming, and reusing agent experience. These mechanisms differ fundamentally along a spectrum of *carrier forms*: from explicit tokenized records (textual reflections, structured workflows, code skills) through continuous latent representations (KV caches, soft prompts, memory tokens) to fully implicit parametric encodings (fine-tuned policy weights, trained reward models). Each carrier form presents a distinct trade-off profile across dimensions of reusability, verifiability, inference cost, and robustness. No universal optimum exists; the suitable carrier depends on the constraints of the downstream task.

This survey organizes the rapidly growing literature on agent learning and memory around a single organizing principle: **experience transformation**—the processes by which agent experience, initially captured as raw interaction logs, is converted across different carrier forms and subsequently reused in decision-making. Rather than surveying the field through the conventional lens of agent components (Memory, Planning, Tool Use) or training techniques (SFT, RAG, RLHF), we treat memory systems, evaluators, and training procedures as manifestations of the same underlying phenomenon: a representation-to-representation pathway in which the same semantic record of agent experience is progressively re-encoded to serve different computational roles.

We introduce a unified carrier taxonomy that classifies experience along the axis of *how experience manifests within model architecture*: **Tokenized** carriers (Narrative and Schematic sub-classes) preserve experience as discrete token sequences addressable through the context window; **Latent** carriers encode experience as continuous vectors or hidden states that directly participate in attention computation; **Parametric** carriers (Policy and Evaluator sub-classes) crystallize experience into neural network weights. These three classes form a continuous spectrum: moving from Tokenized through Latent to Parametric, interpretability and editability decrease while inference efficiency increases.

Within this carrier framework, we identify **seven core transformation pathways** that capture the dominant patterns in the literature:

1. **Narrative → Narrative (P1)** — intra-tokenized abstraction: raw trajectories compressed into reflections, rules, insights, and natural-language skills.
2. **Narrative → Schematic (P2)** — intra-tokenized formalization: natural-language experience structured into code, workflows, knowledge graphs, and executable procedures.
3. **Tokenized → Latent (P3)** — latent compression: discrete token sequences encoded into KV caches, soft prompts, memory tokens, or latent memory banks.
4. **Tokenized → Evaluator (P4)** — evaluator internalization: trajectories distilled into reward models, process reward models, verifiers, and diagnostic critics.
5. **Tokenized → Policy (P5)** — policy internalization: interaction experience absorbed into agent policy weights through imitation, reward-based, or preference-based training.
6. **Evaluator → Policy (P6)** — preference alignment: parametric evaluation signals (reward models, judges) transferred into policy weights via RLHF, DPO, and related methods.
7. **Parametric → Tokenized (P7)** — knowledge externalization: implicit model capabilities read out as synthetic trajectories, demonstrations, and evaluative supervision.

Beyond these atomic pathways, we identify a growing body of work that composes multiple transformations into **composite pipelines**—chains or closed loops in which experience flows across several carrier boundaries and the integration mechanisms between steps constitute the primary methodological contribution. We anatomize three recurring composition patterns: evaluator–policy co-evolution, refinement-mediated policy internalization, and generation-driven composite pipelines.

The remainder of this survey is organized as follows. Section 2 establishes formal preliminaries: the definition of agent experience as a modality-agnostic quadruple, the carrier taxonomy, and the criteria that demarcate genuine experience transformation from mere format conversion. Sections 3 through 6 examine the seven atomic pathways, organized by source carrier: Tokenized-to-Tokenized (P1, P2), Tokenized-to-Latent (P3), Tokenized-to-Parametric (P4, P5), and Parametric-Source Transformations (P6, P7). Section 7 provides a dedicated anatomy of composite pipelines. Section 8 synthesizes cross-pathway findings, comparing trade-off profiles and offering utilization-driven pathway selection guidance. Section 9 concludes with open problems and future directions.
