## Training Method 取值定义（5 类，统一为"目标族"层级）

- **`gen-CE`**（生成式 cross-entropy / MLE）：评估器产出一段**自由形式 token 序列**——verdict 连同 rationale / critique / CoT——以最大化目标序列似然（next-token CE）训练。目标文本来自人工、teacher 或经筛选的自生成。即判官语境下俗称的 "SFT"。
- **`cls-CE`**（分类 cross-entropy，含 BCE）：评估器输出一个**离散正确性/类别标签**（二值或多值）或其概率，**不生成 rationale**，以（二元）交叉熵对标签训练。包含只吐一个受限 verdict token 的 LLM/VLM 判官。
- **`regression`**（MSE）：评估器输出一个**连续标量**——progress / advantage / Q-value / failure probability / graded score——以平方误差对连续目标训练。该目标本身若由 MC 或 TD/GAE 估出，估计法记在 Label Source，不记在此列。
- **`ranking`**（相对序目标）：以候选间的**相对顺序**训练——Bradley-Terry / Plackett-Luce / margin-ranking / pairwise / DPO 式，以及 InfoNCE 式对比——令更优者得分更高，不锚定绝对目标。
- **`RL-PG`**（策略梯度 RL）：评估器**自身作为 policy**、对其生成的判断按 reward 优化（PPO / GRPO / RLOO 及变体），reward 检验 verdict 正确性、格式或下游效果。通常作用于生成式判官，多数前接一段 gen-CE 冷启。

**多值约定**：`A + B` 同时覆盖两种情形——**多阶段**（冷启后接 RL：`gen-CE + RL-PG`）与**多头**（分类头 + 推理解码器：`cls-CE + gen-CE`）。

`TD/GAE` 不是 loss，是 target 估计法，已从本列移出、并入 AgentPRM 的 Label Source（`MC-rollout (TD/GAE)`）。

## 三点说明

**与 Output Form 的相关性（不冗余）**：disc 行落在 `cls-CE` / `regression` / `ranking`（按目标是标签 / 标量 / 相对序三分），gen 行落在 `gen-CE` 或叠加 `RL-PG`。叉乘有信息——同为 gen，GenRM (`gen-CE`) vs J4R (`RL-PG`) 是真区别；同为 disc，Let's Verify (`cls-CE`) / Rewarding Progress (`regression`) / PQM (`ranking`) 也是真区别。哪天要砍宽度，先把 `gen-CE`/`cls-CE` 并成 `supervised`（退回 §5.1.3 的 supervised/preference/RL 三分），比砍整列代价小。

**`cls-CE` 的边界**：Du23、Wen25b 是 LLM/VLM 判官、机制上是对一个受限 verdict token 做 next-token CE，因"目标是类别标签、不带 rationale"归入 `cls-CE`，与 GenRM（生成 CoT，归 `gen-CE`）区分开。判据是**是否生成自由文本**，不是是否用 LM head。

**`RL-PG` 的冷启残留**：S2J、J4R、CTRL、StepWiser、OS-Oracle、RL4F、ECHO 我按论文明述的 RL 主训练标 `RL-PG`，但这类生成式判官多半前接一段格式/SFT 冷启，我未逐篇确认；若较真，其中部分可能应为 `gen-CE + RL-PG`（已确认两阶段的是 DeepCritic、AgentV-RL、WebArbiter、SOLE-R1、GUI-Critic-R1）。BiRM 的 `cls-CE + regression` 取自 §5.1 对双头的描述（backward 正确性头 + forward value 头），非独立读全文核实。要的话我把这 7 篇的冷启逐篇确认一遍。
