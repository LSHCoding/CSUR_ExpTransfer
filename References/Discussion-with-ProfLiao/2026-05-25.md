关于question 1 ：why is experience the right abstraction for organizing the field, as opposed to more established concepts like trajectories, memory, or feedback?

Artificial Intelligence: A Modern Approach（Russell & Norvig，第 4 版）中，Agent（智能体）的定义是 “anything that can be viewed as perceiving its environment through sensors and acting upon that environment through actuators”.

rational agent （研究人工智能方法的核心）的定义：“For each possible percept sequence, a rational agent should select an action that is expected to maximize its performance measure, given the evidence provided by the percept sequence and whatever built-in knowledge the agent has.”


而一次循环，从环境观测，行动，输出，和反馈，构成了 Agent 一次行动的最小完整记录，我们将这个定义为具有最小语义单位的原始经验（针对 question 1）。

关于 question 2：why is carrier-to-carrier transformation the key bottleneck, rather than, say, better retrieval, prompt design, or architectural choices?


在定义了最小语义单位的原始经验之后，我们发现，Experience 会通过转化以不同的形式存在，比如以 token 记录的形式存在，以 参数的形式存在等，我们发现 Experience 主要有三种存在形式：Token，latent 和 Parameter。不同的形式存在不同的优缺点，在具体的场景中会有不同的效果。有时候只是优化单一问题的经验并不能解决问题。比如 Policy 的 Parameter 形式，其推理效率就是比 Token （RAG 形式使用）高，并且能和之前的经验（Parameter） 建立更深层层次的联系（针对 question 2）。
