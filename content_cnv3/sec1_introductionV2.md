# 1. Introduction

从静态大语言模型（LLM、VLM 等）到自主 Agent 的转变，不仅意味着模型能力的扩展，更带来一个根本性变化：Agent 在持续的 experience loop 中运作——在具体决策上下文下尝试任务、产生异构动作（推理轨迹、工具调用、环境控制等）、观察环境返回的客观后果，并可选地接收评价反馈。这些累积的决策证据最初以原始日志（raw trajectory）形式存在。直接保留原始日志并通过检索复用（如 RAG）虽然直观，但在长程 Agent、test-time search 和过程级监督等趋势下于规模上迅速变得昂贵且脆弱：上下文窗口受限阻碍长期保留，检索延迟拖慢决策，单条 episode 的噪声妨碍跨任务泛化。

社区由此发展出多种将原始经验转化为可复用形式的机制。记忆摘要将原始轨迹提炼为自然语言反思与规则（如 Reflexion、ExpeL）；技能归纳将成功轨迹转化为可复用的代码或工作流（如 Voyager 的 skill library）；奖励建模将人类偏好标注固化为参数化评估器；RLHF 和 DPO 将评估信号进一步内化为策略权重；KV cache 复用与 soft prompt 训练将频繁访问的经验压缩为连续隐空间表示；知识蒸馏与自生成训练（如 self-play、SPIN、ReST 系列）将已内化的能力外化为新的训练轨迹。

这些机制各自改进了 agent 系统的某个方面。但一个令人困惑的事实是：在传统的分类体系下，它们被视为互不相关的独立话题。按"组件"（Memory / Planning / Tool Use）组织时，记忆摘要是 Memory 的改进，技能归纳是 Tool 的扩展，RLHF 是 Alignment 的技术——彼此之间的深层关联被模块边界遮蔽。按"技术"（RAG / SFT / RLHF）组织时，摘要和检索同属 RAG 体系，行为克隆和偏好优化分别是 SFT 和 RL 的不同变体——技术标签同样无法揭示这些工作共享的底层逻辑。一个构建 agent 系统的研究者需要同时回答：记忆摘要在什么条件下比 RAG 更有效？RLHF 和 SFT 在将经验内化为能力时各自的适用边界在哪里？技能归纳和知识蒸馏解决的是否是同一个问题？这些知识分散在不同的子领域文献中，缺少一种共同语言来讨论它们之间的关联。

是否存在一个统一的透镜，能够将这些表面异质的工作读作同一个基本动作的不同变体？

本文的论点是：它们确实共享一个深层结构。这个结构是——**将同一份 agent 交互经验从一种载体（carrier）重新表示为另一种载体，以突破源载体的固有 trade-off 限制。** 记忆摘要是经验从冗长的叙事形式到精炼的叙事形式的抽象提纯：保留语义、压缩体积，以应对上下文窗口的物理限制。技能归纳是经验从叙事形式到图式形式的形态转换：将"描述怎么做"变成"可直接执行的结构"，省去每次复用时的重新推理。RLHF 中的奖励建模是经验从 tokenized 形式到参数化评估器的判断力内化：将人类偏好从逐条标注转化为可批量调用的评分函数；随后的 PPO/DPO 是该判断力从评估器参数到策略参数的二次转化：将外部评分信号吸收为模型的生成偏好。记忆摘要、技能归纳、RLHF——这三种在传统分类下分别属于 Memory、Tool、Alignment 子领域的工作——在此视角下显露为**同一种操作**：当经验在当前载体中触及该载体固有的效率/可解释性/可编辑性天花板时，将其语义内容迁移到另一个载体中。我们称这个操作为 **Experience Transformation**（经验转化）。

本文以经验转化为主组织轴，将上述机制系统化为七条基础转化路径——Tokenized 载体内部的语义抽象（Narrative → Narrative）与形式化（Narrative → Schematic）、Tokenized 到 Latent 的隐空间压缩、Tokenized 到 Parametric 的评估器内化与策略内化、Parametric 载体之间的偏好吸收（Evaluator → Policy）、以及 Parametric 到 Tokenized 的知识外化——并识别了将多条路径链式组合的复合模式（composite pipelines）。全文不按"组件"或"技术"分章，而是按转化路径的源载体类型聚合，使每条路径在可解释性、推理效率、可编辑性等维度上的 trade-off 能够直接在分类轴上定位和比较——而非像传统分类那样需要逐篇论文事后提取。

在定位上，本文与几类已有综述形成互补。Agent 架构综述（如 Wang et al., 2024; Xi et al., 2025）按功能模块（Memory / Planning / Tool Use）组织文献，回答"系统由哪些部分组成"；记忆系统综述（如 Zhang et al., 2025）聚焦 Tokenized 载体内的存储与检索机制，回答"经验如何被存取"；对齐综述追踪偏好优化技术的演进，回答"如何让模型行为与人类价值一致"；过程奖励模型综述（如 Zheng et al., 2025）深入单一载体（Evaluator Parameters）的设计空间。这些综述各自在特定维度上提供了深度覆盖，但没有一部以**经验在不同载体之间的迁移机制**为主线。本文的独特贡献恰在于此：将上述分散的话题统一到"同一语义内容在不同表示形式间的转化"这一框架下，使跨子领域的结构性比较成为可能。

在方法论上，本文的纳入标准要求数据同时满足两个条件：（1）具备序贯决策语义，可映射为经验四元组 $e = (c, a, o, f)$；（2）动作 $a$ 属于 LLM-based system 的异构动作空间（推理轨迹、工具调用、规划分解、环境控制等）。排除范围包括：静态语料预训练（无决策过程语义）、单步分类/标注的 SFT（非异构动作空间）、纯模型蒸馏 Parametric → Parametric（经验语义链断裂），以及非 LLM-based 系统。经验来源不限——agent 自身轨迹、人类专家示范、teacher model 合成数据——判定依据底层机制而非论文的自我定位。

本文的主要贡献是提出了一个以经验转化为轴心的统一分析框架，并在此框架下完成了三项工作：（1）系统梳理了七条基础转化路径的机制特征、代表性方法与 trade-off 画像；（2）识别并解剖了复合路径的三种衔接模式，填补了单路径分析无法覆盖的集成机制空白；（3）提供了跨路径的 trade-off 对比与场景化方法选型参考。全文组织结构如下：Section 2 建立概念体系并给出选择经验转化作为组织轴的完整论证；Sections 3–6 按源载体类型分组展开七条单路径的文献分析；Section 7 讨论复合路径；Section 8 做跨路径综合与利用分析；Section 9 讨论开放问题与未来方向。
